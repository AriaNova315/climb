## 一、产品概述

### 1.1 产品定位

一个面向攀岩爱好者的北京攀岩馆信息聚合地图网页，帮助用户快速了解全市岩馆分布、类型与基本信息，降低找馆成本。

### 1.2 目标用户

- 北京的攀岩初学者：不知道附近有哪些馆、馆有什么区别
- 有经验的攀岩者：寻找特定类型（抱石/难度/综合）的场馆
- 外地来北京的攀岩者：快速定位城区内可去的馆

### 1.3 核心价值

> 一张图，看清北京所有攀岩馆的位置与类型。

## 四、页面与交互设计

### 4.1 唯一页面（MVP）：地图主页

```
┌─────────────────────────────────────────┐
│  北京攀岩馆地图          [图例 Key]      │  ← 顶部导航栏（简洁）
├─────────────────────────────────────────┤
│                                         │
│         [北京地图 SVG]                  │
│                                         │
│   各区域用不同背景色区分                 │
│   岩馆位置用彩色圆点标记                 │
│   点击圆点 → 触发弹窗                   │
│                                         │
└─────────────────────────────────────────┘
```

### 4.2 岩馆信息弹窗（核心交互）

**触发方式**：点击地图上的岩馆圆点标记

**弹窗内容**：

```
┌──────────────────────────┐
│ [类型标签]  攀野          │  ← 馆名 + 类型色标
│ ─────────────────────── │
│ 📍 昌平区 xxx            │  ← 地址
│ ⏰ 10:00 - 22:00        │  ← 营业时间
│ 💰 单次 80-120 元        │  ← 价格范围
│ 📞 010-xxxxxxxx         │  ← 电话
│                          │
│ [抱石] [难度] [培训]      │  ← 标签
│                          │
│      [查看详情 →]         │  ← 第二阶段跳转入口（MVP 可灰显）
│                          │
│                      [×] │  ← 关闭按钮
└──────────────────────────┘
```

**弹窗行为**：
- 点击弹窗外区域 → 关闭弹窗
- 点击 `×` → 关闭弹窗
- 点击另一个标记 → 当前弹窗关闭，新弹窗打开
- 弹窗出现位置：优先显示在标记点旁边，靠近边缘时自动翻转方向

### 4.3 图例

固定显示在地图右上角（或左上角），内容：

```
图例 Key
● 综合馆（抱石+难度）
● 抱石馆（仅抱石）
● 难度馆（仅难度）
```

## 五、技术设计

### 5.1 技术栈

| 层次 | 技术选型 | 选型理由 |
|------|----------|----------|
| 框架 | Next.js 14 (App Router) + TypeScript | 路由扩展方便，Vercel 一键部署 |
| 地图（MVP） | SVG（Figma 直接导出） | 零依赖，完全还原设计，开发快 |
| 样式 | Tailwind CSS | 与设计稿快速对应 |
| 弹窗组件 | 自定义（无第三方 UI 库） | 保持轻量，样式完全可控 |
| 数据 | 本地 JSON 文件 | MVP 无需后端 |
| 部署 | Vercel | Next.js 官方推荐，免费额度足够 |

### 5.2 项目结构

```
climb/
├── app/
│   ├── page.tsx              # 地图主页
│   ├── layout.tsx
│   └── gym/
│       └── [id]/
│           └── page.tsx      # 岩馆详情页（第二阶段）
├── components/
│   ├── Map.tsx               # SVG 地图主组件
│   ├── GymMarker.tsx         # 单个岩馆标记点
│   └── GymPopup.tsx          # 岩馆信息弹窗
├── data/
│   └── gyms.json             # 岩馆数据
├── public/
│   └── map.svg               # 从 Figma 导出的地图 SVG
└── types/
    └── gym.ts                # 数据类型定义
```

### 5.3 SVG 地图实现方案

1. 从 Figma 导出完整地图为 SVG 文件
2. 地图背景（区域色块）作为静态底图
3. 岩馆标记点作为独立的可交互 SVG 元素，绑定 `onClick` 事件
4. 点击后更新 React state，渲染对应弹窗

## 七、待确认事项

在开始开发前，需要你确认以下问题：

1. **岩馆数据来源**：是否有完整的岩馆名单和基础信息？还是需要我从地图设计图中逐一整理？

2. **Figma 导出权限**：SVG 方案需要从 Figma 导出地图文件，你有导出权限吗？还是希望我基于截图重建 SVG？

3. **弹窗字段优先级**：MVP 弹窗中"价格范围"和"电话"是否必须？如果暂时没有数据，是否可以先显示"暂无"占位？

4. **设计色值**：能否从 Figma 中导出或告知各区域/标记点的精确颜色值？

5. **部署平台**：是否有 Vercel 账号？或者是否有其他指定的部署平台？

6. **域名**：MVP 阶段用 Vercel 默认域名即可，还是有自定义域名需求？
